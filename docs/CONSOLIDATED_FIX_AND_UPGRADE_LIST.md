# דוח מרוכז — תיקונים ושדרוגים מומלצים (VIPO Agents)

מסמך זה **מסכם** את ממצאי הסקירות בצ’אט: ארכיטקטורה, אבטחה, דפים, CI, ו-Git.  
**סטטוס “מושלם לפרסום נרחב”** דורש גם **בדיקות ידניות** (מסלול רכישה, תשלום, מובייל) — אלה מופיעים בסוף.

---

## א. כבר טופל בקוד (לידיעתכם)

| נושא | מה נעשה |
|------|---------|
| מחיקת הודעות מ־`read-all` | הוסרה מחיקה אוטומטית; נשאר סימון כנקרא + דוח `cleanup` ללא מחיקה |
| URL שגוי `/businesss` | הפניות קבועות ב־`next.config.js` ל־`/business`; הוסר דף העתק ב־`app/businesss/` |
| שער אוטומטי לפני יציאה | `npm run prelaunch:gate` — lint, ניקוי `.next`, build, typecheck, קבצים קריטיים → `reports/pre-launch-gate.json` |
| מלאי דפים (App Router) | `docs/PRE_LAUNCH_PAGE_AUDIT.md` — ~140 דפים + צ’ק־ליסט ידני |

---

## ב. חובה לטפל לפני / במהלך פרסום נרחב (עדיפות גבוהה)

### 1. בדיקות ידניות (“Go live”)

- [ ] **מסלול מכירה מלא**: קטלוג / מוצר → עגלה → checkout → ספק תשלום (PayPlus וכו') → הצלחה / ביטול / מייל אם קיים  
- [ ] **מובייל + RTL** (אם אתם מוכרים בעברית)  
- [ ] **דפי אמון**: `/privacy`, `/terms`, `/returns-policy` — תואם מציאות (עוגיות, שיווק, וואטסאפ)  
- [ ] **אין חשיפת dev/demo בפרודקשן** (למשל `/checkout-demo`, נתיבי `dev/*` — לוודא חסימה או הסרה מפרוד)

### 2. SEO ומסע פרסום

- [ ] **שני דפי בית**: `/` (Marketplace) ו־`/homepage` (תוכן אחר) — **להחליט canonical אחד** למודעות וקישורים; למנוע כפילות אינדקס אם לא נדרש  
- [ ] **Sitemap / robots** — לוודא שרק URL ציבוריים רצויים נסרקים  

### 3. אבטחה וגבולות

- [ ] **API ללא middleware גלובלי**: נתיבי `app/api` לא עוברים דרך `middleware.js` — כל handler חייב אימות/tenant בעצמו; לעשות **סקירת דגימה** לנתיבים: תשלום, webhooks, מחיקות admin  
- [ ] **Rate limit בתוך זיכרון** (`lib/rateLimit.js`): ב־**מספר instances** (Vercel) המגבלה **לא משותפת**. לשדרג ל־**Redis / Upstash** לרגישים (login, register, checkout session) אם יש עומס/התקפות  

### 4. איכות קלט (Zod)

- [ ] יש **מאות** נתיבי API; רק **מיעוט** עם Zod נראה בדגימה — להרחיב **בנתיבים הרגישים ביותר** (כסף, הרשאות, bulk, ייבוא)

## ג. חשוב (לא חוסם launch, אבל “חובת מערכת בשלה”)

| נושא | הסבר קצר |
|------|-----------|
| **CI** | Nightly מריץ בעיקר `test:api`. מומלץ PR: `lint` + `typecheck` + `test:api` (ובהמשך e2e נקודתי) |
| **`experimental.missingSuspenseWithCSRBailout: false`** | מכסה אזהרות build; עדיף **לתקן דפים נקודתית** מאשר להשאיר לנצח |
| **בדיקות E2E ממוקדות** | 3–5 תרחישי Playwright: בית → מוצר → עגלה → checkout (בלי תשלום אמיתי אם אפשר) |
| **Git / מקור אמת** | אם עדיין יש תיקייה ללא remote תקין — **מקור אחד** + `push` סדיר כדי לא לאבד שינויים |

---

## ד. סיכון תפעולי (לא “באג”, אבל להכיר)

| נושא | הערה |
|------|------|
| **instrumentation + auto-sync** | אופציוני (`AUTO_SYNC_ENABLED`); בפרודקשן לוודא שלא מושך/דורס נתונים בטעות |
| **כמות קוד גדולה** | ~280+ קבצי API + ~140 דפים — תחזוקה דורשת תהליך (שינויים קטנים, PR ממוקדים) |

---

## ה. סיכום מהיר — “מה לתקן עכשיו”

1. **ידני**: מסלול רכישה + מובייל + מסמכי אמון + וידוא שאין dev בפרוד.  
2. **מוצר/SEO**: החלטה על `/` מול `/homepage` וקמפיינים.  
3. **אבטחה**: דגימת API קריטיים; תכנון rate limit מבוזר אם יש פריסה multi-instance.  
4. **איכות**: Zod בנתיבי סיכון גבוה.  
5. **אוטומציה**: להריץ לפני כל RC: `npm run prelaunch:gate` ולקרוא `reports/pre-launch-gate.json`.
